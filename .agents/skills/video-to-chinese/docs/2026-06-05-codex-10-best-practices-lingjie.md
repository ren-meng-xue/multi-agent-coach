# 把 Codex 用到极致：官方最佳实践 10 个重点

## 一句话总结

OpenAI 官方发布《Getting the most of Codex》，灵界 AI Talk 结合实操讲解了 10 个核心功能要点——从线程管理、目标驱动、自动化到共享记忆——帮你把 Codex 从代码工具升级为真正能跑工作流的通用 Agent。

## 核心观点

1. **线程（Threads）是上下文管理的基本单元**：给活跃线程编号或置顶；上下文快溢出时先让 Codex 生成"交接文档"，再开新窗口——而非直接新开、丢失进度。

2. **语音输入能捕捉你还没想清楚的想法**：打字会过滤掉模糊的念头，语音则全量输入给 AI；配合规划模式（Planning Mode），可以先让 AI 整理意图再执行，节省 token 和返工成本。

3. **引导与排队（Steering & Queuing）适合长任务中途插入**：任务执行中可随时"插入"新指令（立即合并）或让后续指令"排队"（完成当前任务后自动触发）；不需要打断当前任务。

4. **Goal（目标）模式是复杂项目的关键**：区别于"给步骤"，Goal 模式让你提供"目标 + 可衡量的完成标准"，Codex 会持续自我回顾目标直到完成；与 GPT5.5 的 goal-oriented 范式一脉相承。

5. **插件与 MCP 扩大了 Codex 的触达范围**：内置 Notion、Gmail、Figma、Remotion、Canva 等插件；MCP 服务器会在对话中按需自动连接，无需手动配置；Computer Use 支持 Windows，但 Mac 体验更流畅（Windows 下 Codex 操控系统时用户无法同时操作）。

6. **三个设置值得关注**：① App Snapshot（双击 Command 键快速截图）；② Enable Memory（实验性，让 Codex 持续理解你的使用习惯和身份）；③ 深色主题护眼。

7. **随时随地工作（Work from Anywhere）= 手机扫码远程监控任务**：在外出时可通过 ChatGPT App 扫码连接桌面 Codex，查看长任务进度并发送补充指令；IP 不稳定时连接可能失败。

8. **Automation（自动化）是"心跳机制"**：可直接通过对话生成定时任务，例如每日自动收集 Gmail 摘要；这是 A2A（Agent-to-Agent）逻辑的雏形——AI 先读邮件、判断优先级，人只看结果再决策。

9. **侧边栏（Side Panel）让你不离开对话就能预览产出物**：点击线程中的 MD 文档、PDF、图片，内容直接显示在右侧；作者建议尽量用 MD 文件而非 PDF/Excel，因为 MD 对机器更友好、体积更小。

10. **共享记忆（Shared Memory）是跨线程、跨 AI 协作的核心**：不要依赖大模型自动记忆——要定期让 Codex 将上下文、项目理解、技能抽象写入记忆文档；这份文档可跨线程激活，也可粘贴给 GPT（因 GPT 无法读本地文件），实现 Codex + GPT 的知识共享和规划-执行循环。

11. **"从代码向外扩展"（From Code Outward）= Codex 已是通用 Agent**：设置中有"编程模式"和"日常工作模式"之分，前者返回更多技术细节，后者更口语化；作者认为差别不大，Codex 早已不是 IDE 工具，按需选择即可。

## 可执行建议

- **线程快溢出时，让 Codex 先写"交接文档"**：告诉它"上下文已经很慢，生成一份 MD 摘要文档，让另一个窗口能快速接手"，再开新线程
- **复杂任务先开 Planning Mode**：避免跑完大半发现方向错了；检查规划路径后再执行，省 token 省时间
- **用 Goal 替代"给步骤"**：不要写"第一步做X、第二步做Y"，而是写"目标是Z，完成标准是……"，让 Codex 自主规划并持续执行
- **给活跃线程编号或置顶**：多任务并行时，编号（如 "01-内容工厂"）方便快速切换
- **定期让 Codex 写记忆文档**：把项目理解、使用习惯、技能抽象写成 MD 文件，跨线程激活时无需重复解释背景
- **尝试用 Automation 自动化重复性信息收集**：通过对话生成定时任务，接入 Gmail、Notion 等插件，让 AI 先过滤再汇报
- **优先用 MD 文件存储产出物**：比 PDF/Excel 更小、更机器友好，侧边栏也能直接预览

## 关键术语

| 术语               | 中文解释                                                         |
| ------------------ | ---------------------------------------------------------------- |
| Durable Threads    | 持久线程，Codex 的对话主题单元，每个线程有独立上下文             |
| Steering           | 引导，任务执行中途插入新指令，立即与当前任务合并                 |
| Queuing            | 排队，将后续指令预先排入队列，当前任务完成后自动触发             |
| Pursue Goals       | 追求目标，Codex 的长任务模式，AI 持续自我回顾直到达成可衡量目标  |
| Computer Use       | 计算机使用，Codex 模拟人类操控桌面的能力，Mac 支持更完整         |
| App Snapshot       | 应用截图，双击 Command 键快速截取当前页面内容                    |
| Enable Memory      | 启用记忆，实验性功能，让 Codex 跨对话记住用户习惯和身份          |
| Work from Anywhere | 随时随地工作，手机端扫码远程监控和控制桌面 Codex 任务            |
| Automation         | 自动化/心跳机制，设置定时任务让 Codex 周期性执行特定工作         |
| Side Panel         | 侧边栏，点击产出物在右侧预览，无需离开对话窗口                   |
| Shared Memory      | 共享记忆，将项目上下文写入 MD 文件，实现跨线程、跨 AI 知识共享   |
| A2A                | Agent-to-Agent，智能体间通信，如 Codex 代读邮件后再让人决策      |
| MCP                | 模型上下文协议，Codex 连接外部工具（Notion、Gmail 等）的标准接口 |
| From Code Outward  | 超越代码，Codex 从 IDE 工具进化为通用 Agent 的定位转变           |
