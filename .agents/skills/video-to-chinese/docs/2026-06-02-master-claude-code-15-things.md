# 掌握 Claude Code 80% 功能：只需学会这 15 件事

## 一句话总结

Claude Code 不是普通聊天机器人——它能读文件、写文件、执行命令；掌握计划模式、上下文管理、CLAUDE.md、Skills、Hooks、MCP 等 15 个核心概念，你就能覆盖 80% 的实际使用场景，从会用工具进化到用 Claude Code 驱动业务。

## 核心观点

1. **Claude Code 是执行者，不是顾问**：ChatGPT 只能给建议，Claude Code 可以直接在你的机器上读、写、跑命令，让它干活、审核结果即可。
2. **上下文是绝对的命门**：token 越多模型越健忘——约 10,000 token（约 7,500 词）后就丢失 50% 的信息。所有高级功能（plan mode、/compact、Skills、CLAUDE.md）本质都在解决"如何在正确时间给正确上下文"这个问题。
3. **CLAUDE.md 应该是一个目录，不是一本百科全书**：保持 <200 行，指向其他文件让 Claude 按需加载，而不是每次启动堆满上下文。
4. **Skills > Slash Commands**：Skills 通过渐进式披露实现"先看标题，再按需加载全文"，配合 reference 文件可以把复杂度打散，避免上下文膨胀。
5. **从"技能"到"业务系统"的跃迁**：不要只建孤立 Skill，要按"待完成的工作"（jobs-to-be-done）建模块化 Skill 链，配合共享业务上下文文件夹，形成 Agentic Operating System。

## 可执行建议

- **每次非简单任务都用 Plan Mode**：双击 Shift+Tab，让 Claude 先出计划再动手，避免返工
- **安装 CC-Notify**：实现异步工作——启动任务后离开，需要审批时被通知拉回
- **保持 CLAUDE.md < 200 行**：只放核心规则，其余信息放独立文件，在 CLAUDE.md 中用路径引用
- **不相关任务之间及时 /clear**：防止上下文腐化导致质量下降
- **建 Skill 而非重复粘贴提示词**：用 Anthropic 的 skill-creator skill 生成，确保 < 200 行 + references 文件夹结构
- **用 Hooks 加载必须每次都出现的上下文**：比如 SessionStart Hook 加载项目基础信息，不依赖 LLM 判断
- **按 jobs-to-be-done 建 Skill 链**：拆分研究、写作、格式化、审核为独立模块，可自由组合
- **记忆系统优先做到 L3（语义搜索）**：这是大多数业务用户的 80/20 分水岭——让 Claude 按含义而非关键词查找信息

## 关键术语

| 术语                   | 中文解释                                                         |
| ---------------------- | ---------------------------------------------------------------- |
| Plan Mode              | Claude Code 的计划模式，双击 Shift+Tab 进入，先规划再执行        |
| Context Rot            | 上下文腐化，token 越多模型越容易忘记前面内容的现象               |
| /compact               | 压缩命令，在不切换话题时压缩对话历史，保留关键信息               |
| Resume                 | 会话恢复，让 Claude 记住上一轮对话的完整上下文                   |
| Progressive Disclosure | 渐进式披露，Skill 先加载名称/描述，触发后才加载完整内容          |
| MCP                    | 模型上下文协议（Model Context Protocol），连接外部工具的标准接口 |
| Agentic OS             | Agentic 操作系统，一套共享业务上下文 + 模块化 Skill 的框架       |
| Hooks                  | 钩子，在特定事件（如会话开始、工具调用前）必定触发的确定性脚本   |
| Jobs-to-be-done        | 待完成工作，一种产品思维——按业务工作拆分功能，而非按功能列表     |
