## 一句话总结

模型本身不是 Agent 的安全边界——被调用的 API 才是；必须在权限、审计、回滚三个维度做架构级设计，仅靠模型层控制会被 prompt 注入击穿。

---

## 核心观点

**1. Agent 风险与 LLM 有本质区别**
LLM 的风险在语义层（说错话），Agent 的风险在行动层（做错事）。Agent 能查数据、删记录、发邮件，传统输入过滤和输出审核管不了行动层，必须从架构层面控制。

**2. 最小权限 = 刚好够用，不是越小越好**
给 agent 管理员权限风险太大，直接禁止什么都干不了。正确做法是给恰好满足任务需要的最小权限，例如只读查询权限。

**3. 权限设计的五个控制面**

- **工具权限分层**：按风险等级（查询 < 修改 < 删除 < 支付）配置不同策略，高风险必须人工确认+回滚，严重风险无回滚能力不允许上线
- **工具规格定义**：每个工具的操作范围、作用域、副作用、是否需要回滚写成"员工手册"，调用前边界明确
- **上下文裁剪**：只给 agent 当前任务需要的信息，避免因上下文过多而接触到不应访问的敏感数据
- **执行隔离**：模型只负责生成指令，执行层负责校验和执行，且执行层不读原始输入，只处理解析后的结构化指令（No Raw Input Principle）
- **API 层强制执行**：权限校验必须在 API 层实施，不能依赖模型判断，因为模型可被 prompt 注入攻击绕过

**4. 审计日志的目的是支持响应，不只是记录**
四步骤闭环：发现行动 → 范围风险可视化 → 活跃度追踪 → 循环节奏（周/两周/月/季度）。真实案例：某企业审计发现 100+ AI agent，其中两个完全无主。

**5. 回滚必须分级设计**

- 低风险（只读）：无需回滚
- 中风险（insert/update）：记录变更日志，支持人工回溯
- 高风险（删除/发邮件）：人工确认 + 回滚能力
- 严重风险（系统配置变更/代码执行）：无回滚能力则禁止执行

**6. 短期令牌控制外部工具影响范围**
外部工具访问使用短期令牌，作用域限定为特定操作，工作流结束后自动失效，出问题时影响可控。

**7. 默认安全原则（来自真实漏洞教训）**
高风险能力默认关闭，需要显式授权才能启用。CVE-2024-27966（LangFlow 平台 CSV Loader 节点）因安全开关硬编码为 true，导致间接 prompt 注入→结构化工具调用→策略缺失→代码执行→权限扩大的完整攻击链。

**8. 权限蔓延的治理**
根因是"以后再收紧"的承诺从未兑现。解法：人员离职自动触发权限重审；工具规格定义把权限写死；没有规格的工具默认禁止。

## 可执行建议

1. **工具注册时强制填写规格**：操作范围、作用域、副作用、回滚方式缺一不可，无规格默认禁止上线
2. **权限校验下沉到 API 层**：在每个工具的执行入口加独立校验，不依赖上层模型判断
3. **每次部署后立即做 agent 清查**：列出所有 agent 及其授权范围，标注授权人是否在职
4. **审计节奏写进 SOP**：每周审查权限范围，每两周撤销陈旧令牌，每季度全面复查
5. **高风险操作加人工确认节点**：删除、发邮件、代码执行等操作上线前设计回滚路径，无路径不上线
6. **人员离职流程触发权限重审**：将 agent 权限纳入离职 checklist，自动降权或撤销

## 关键术语

| 术语                   | 解释                                                   |
| ---------------------- | ------------------------------------------------------ |
| 最小权限原则           | 给 agent 刚好满足任务的最小权限，不多不少              |
| 执行隔离               | 模型生成指令，执行层独立校验后执行，两者职责分离       |
| No Raw Input Principle | 执行层不直接处理原始输入，只处理经过解析的结构化指令   |
| 工具规格定义           | 描述工具操作范围、作用域、副作用、回滚需求的元数据     |
| 间接 prompt 注入       | LLM 读取外部内容时，恶意指令被当作高优先级规则处理     |
| 策略裁决点             | 独立于模型的权限判断机制，防止模型被注入后绕过安全检查 |
| 默认安全原则           | 高风险能力默认关闭，需显式授权启用                     |
| 权限蔓延               | 初始发布权限过宽、后续未收紧导致权限范围持续扩大的现象 |
