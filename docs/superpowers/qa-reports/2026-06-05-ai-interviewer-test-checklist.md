## 维度二：质量验证

### Q1. 问题相关性

| #    | 测试项                               | 测试方法                                     | 通过标准                                     | 优先级 |
| ---- | ------------------------------------ | -------------------------------------------- | -------------------------------------------- | ------ |
| Q1-1 | 问题与 JD 岗位职责强相关             | LLM-as-judge：将问题 + JD 发给评审 LLM，打分 | 相关性评分 ≥ 4/5（5 道题平均）               | 高     |
| Q1-2 | 问题结合候选人简历背景定制化         | LLM-as-judge + 人工抽查                      | ≥ 60% 的问题包含对候选人具体背景的指向性引用 | 高     |
| Q1-3 | 不提问候选人简历已明确给出答案的内容 | LLM-as-judge                                 | 无"重复确认简历已有信息"的无效问题           | 中     |

---

### Q2. 追问质量

| #    | 测试项                        | 测试方法                               | 通过标准                                                      | 优先级 |
| ---- | ----------------------------- | -------------------------------------- | ------------------------------------------------------------- | ------ |
| Q2-1 | 能识别浅层/模糊回答并触发追问 | 手动：发送"我负责了整个项目"类浅层答案 | 面试官追问具体细节，不接受作为完整答案                        | 高     |
| Q2-2 | 追问分层：技术细节层          | 手动 + LLM judge                       | 追问中出现"具体怎么实现的"/"用了什么技术方案"类问题           | 高     |
| Q2-3 | 追问分层：行为逻辑层          | 手动 + LLM judge                       | 追问中出现"你当时为什么选择这个方案"/"遇到阻力怎么处理"类问题 | 高     |
| Q2-4 | 追问分层：量化结果层          | 手动 + LLM judge                       | 追问中出现"结果提升了多少"/"影响了多少用户"类量化问题         | 中     |
| Q2-5 | 追问不超过 3 层（不无限深挖） | LLM judge + 脚本计数                   | 单题追问轮次 ≤ 3，超过 3 轮自动推进至下一题                   | 中     |

### Q3. 语气与中立性

| ---- | -------------------------------------------- | -------------------------- | --------------------------------------------------------- | ------ |

## LLM-as-Judge Prompt 模板（可复用）

```
你是一名专业面试质量评审员。
输入：
- 岗位 JD：{jd}
- 候选人简历摘要：{resume}
- 面试官问题/回复：{message}

请从以下维度打分（1-5分），并给出 1 句理由：
1. 问题与 JD 相关性
2. 语气中立性（5=完全中立，1=有明显引导/评价）
3. 追问有效性（如适用）

输出 JSON：{"relevance": N, "neutrality": N, "followup": N, "reason": "..."}
```
