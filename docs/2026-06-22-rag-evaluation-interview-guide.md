# 如何评估 RAG 系统效果（面试题全解析）

## 一句话总结

RAG 评估不能用 BLEU/ROUGE，要用 RAGAS 三维度（忠实度/答案相关性/上下文相关性）+ 三层分层测试（retriever → generator → 端到端）+ LLM-as-judge 偏差缓解，面试回答到第五层才是优秀。

## 核心观点

1. **传统 NLP 指标（BLEU/ROUGE）在 RAG 上天然失效** — 三个原因：只看字面重叠不懂语义、完全无视上下文来源、无法检测幻觉。RAG 的核心风险恰好是"看起来有据可依但实际在胡说"，而 BLEU 对此毫无感知。

2. **RAGAS 三大核心维度不需要人工标注** — 全部由 LLM 自动打分：
   - **Faithfulness（忠实度）**：把答案拆成独立声明，逐条核查是否有检索文档支持，计算"被支持声明数 / 总声明数"。注意：忠实度 ≠ 正确性，文档写错了答案照实复述仍得满分。
   - **Answer Relevance（答案相关性）**：让 LLM 看着答案反向生成"这在回答什么问题"，用 embedding 余弦相似度衡量与原问题的距离，间接度量是否答非所问。
   - **Context Relevance（上下文相关性）**：从检索文档中挑出真正有用的句子，计算"有用句子 / 总句子"，评估检索器是否在浪费 token。

3. **LLM-as-judge 是当前最大争议点，也是面试追问的核心** — NVIDIA 研究（54 个 LLM × 2000 条 RAG 输出 × 3 名人类标注者）显示，多数主流 LLM 与人类判断方向一致（相关性 >0.8），但三个已知偏差不可忽视：位置偏差（换顺序判断反转）、冗长偏差（长答案被偏好）、自我偏好（模型给自己的输出打高分，且有家族偏好）。

4. **LLM-as-judge 偏差的三类缓解方案** — 不是将就，是约束条件下的最优解：①工程手段（AB 双向评分取平均、评分标准加简洁性维度、用不同家族模型当裁判）；②微调专用小裁判模型（Stanford URM 思路）；③人机混合（自动初筛 + 高风险 case 人工审核）。

5. **三层分层测试是真正的工程方法论** — RAGAS 三指标解决自动化打分问题，但不能定位"问题出在哪一层"。三层评估才能做到诊断：
   - **Layer 1：单独测 retriever**，用 Precision@K、Recall@K、MRR、NDCG 等传统信息检索指标，不涉及 LLM。
   - **Layer 2：单独测 generator**，绕过 retrieval，直接喂"完美文档"，排除检索干扰，定位提示词或 LLM 能力问题。
   - **Layer 3：端到端测试 + 诊断矩阵**，用三层指标联合分析定位根因。

6. **黄金测试集是团队最宝贵的工程资产** — 分层测试依赖它：每条包含问题 + 正确文档 + 标准答案，需覆盖常见问题、多文档综合题、知识库无答案题、信息矛盾题，一般 20~50 条起步，真实失败 case 要持续积累进去。

7. **RAG 有七个学术定义的失败模式，没有单一指标能全覆盖** — ①知识库无答案 ②相关文档排名太低未进 top-k ③上下文被截断 ④答案在文档里但 LLM 未提取出来 ⑤格式不对 ⑥力度不当 ⑦回答不完整。这是多指标组合 + 分层测试的根本原因，面试中主动提出会非常加分。

## 可执行建议

- **面试现场**：不要一上来就说"用 RAGAS 打分"，按五层框架展开，至少说到第三层（LLM-as-judge 局限），才算及格。
- **选 LLM 裁判**：裁判和被评估模型要用不同家族（如生成用 GPT，评估用 Claude），避免家族偏好。
- **AB 双向评分**：每次用 LLM 比较两个答案时，把顺序换一次各评一次取平均，5 分钟成本消除位置偏差。
- **黄金测试集要包含"无答案题"**：知识库里根本没答案是七大失败模式之一，测试集里必须有这类题，否则无法暴露 hallucination 场景。
- **DeepEval 优先用于 CI/CD 门禁**：原生集成是其核心优势，每次发布前自动跑回归评估，防止 prompt/模型变更引入回退。
- **实际工程中定制测试集胜过框架**：RAGAS/DeepEval 做快速初筛，但积累真实失败 case 的领域测试集才是最宝贵的团队资产，不可替代。

## 关键术语

| 术语                              | 说明                                                |
| --------------------------------- | --------------------------------------------------- |
| RAGAS                             | RAG Assessment 框架，提供无需人工标注的自动评估指标 |
| Faithfulness（忠实度）            | 答案中的声明有多少比例有检索文档支持                |
| Answer Relevance（答案相关性）    | 答案与原始问题的相关程度，用反向问题生成间接度量    |
| Context Relevance（上下文相关性） | 检索到的内容中真正有用的比例，评估检索器质量        |
| LLM-as-judge                      | 用 LLM 替代人工来评估 LLM 输出质量的范式            |
| 位置偏差（Position Bias）         | LLM 裁判对同样内容因出现顺序不同而给出不同评分      |
| 自我偏好（Self-Preference）       | LLM 倾向于给自己或同家族模型的输出打更高分          |
| 黄金测试集（Golden Test Set）     | 人工标注的标准数据集，包含问题/正确文档/标准答案    |
| Precision@K                       | top-k 检索结果中相关文档的比例                      |
| Recall@K                          | 所有相关文档中被检索到的比例                        |
| MRR（Mean Reciprocal Rank）       | 第一个相关文档的排名倒数的均值                      |
| NDCG                              | 考虑排序位置的综合质量指标                          |
| DeepEval                          | RAG/LLM 评估框架，以 CI/CD 集成为核心特性           |
