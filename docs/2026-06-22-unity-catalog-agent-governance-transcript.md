Unity Catalog 本质上是你的治理基础。Unity Catalog 是一个集中式数据目录，提供访问控制、审计追踪、数据血缘追踪、数据质量监控，以及更多 Databricks 功能。

工作区环境——由于你将在 Databricks 工作区中操作，最好了解 Databricks 工作区是一个团队编写代码的环境，运行笔记本并访问数据，这是与团队成员协作的理想方式。Unity Catalog 为所有工作区提供统一治理。

**[00:01:02]**

在使用 Unity Catalog 时，需理解其结构和元存储（Metastore）。元存储是 Unity Catalog 中数据的顶级容器。每个元存储注册数据和 AI 资产的元信息，以及管理其访问权限。每个元存储提供三层命名空间：首先是目录（Catalog），其次是模式（Schema），然后是表/视图，通过卷和函数组织数据。

现在让我们深入了解每个部分的含义。我们有数据库对象——Databricks 主要使用两种可安全访问的对象存储数据：首先是结构化表格数据，其次是非结构化非表格数据存放在卷（Volumes）中。

你还可以从表格和卷创建向量数据库，包括向量列和卷。如果你使用 Databricks 向量搜索创建向量索引，这些向量数据库将继承 Unity Catalog 的所有安全功能。Catalog 的安全审计能力适用于所有 AI 检索应用。
