# 你只用了 Claude Code 10% 的功能（认真的）

来源：https://www.youtube.com/watch?v=nPkoRJMIgmk
作者：Simon Scrapes
日期：2026-06-17

---

## 一句话总结

Claude Code 承诺的"后台自主运行"需要四个功能组合才能真正兑现：Auto Mode + /goal、/effort + 计划文件、VPS + tmux 持久化、以及 Connectors/浏览器自动化。

---

## 核心观点

**1. 大多数人只用到了 Claude Code 的 10%**

绝大多数用户依然在"一次一条提示词"地手动驱动 Claude Code，与切换工具之前做的事情没有本质区别。真正能兑现"自主运行"承诺的功能都存在，只是没被打开。

**2. 自动模式（Auto Mode）是消除权限弹窗的关键**

按两次 Shift+Tab 开启 Auto Mode，后台分类器自动放行安全操作，只在真正有风险时（如删除文件）才暂停询问，消除约 90% 的手动授权。

**3. `/goal` 让 Claude 持续工作直到真正完成**

普通提示词由 Claude 自己决定何时"完成"，往往过早结束。`/goal` 强制 Claude 一轮接一轮运行，直到用户定义的完成条件为真——由第二个独立的审核 agent 验证，无法自行宣布完成。

**4. `/loop` + `/routines` 让重复任务全自动定时执行**

`/loop` 设置短期（三天内）的运行节奏，`/routines` 用于每周定时任务。二者与 `/goal` 配合，形成"定时触发 → 自动运行 → 达成标准才停止"的完整闭环。

**5. `/effort` 滑块和计划文件是保证大任务质量的两个抓手**

质量下降有两个根因：上下文腐烂（context rot）和推理算力不足。`/effort` 提高思考 token 配额（low → max → Ultra Code），计划文件放入项目文件夹内（而非临时目录）确保长会话中不丢失方向。

**6. Ultra Code = 动态多 agent 工作流**

Ultra Code 不只是"多想一会儿"，它让 Claude 为当前任务自行设计多 agent 工作流：扇出子 agent 分工研究 → 汇总 agent 综合结果 → 验证 agent 质量审核，每个 agent 拥有独立上下文窗口，无腐烂，由主对话窗口编排协调。

**7. 手机远程派发任务需要 VPS + tmux 才能真正持久**

内置的 Remote Control 和 Channels 功能本身存在"断线即停"的问题。解决方案：在 $15/月 VPS 上运行 Claude Code，用 tmux 保持会话持续存活，再通过 Channels 连接 Telegram/Discord，实现手机随时派发任务并走开。

**8. Connectors 和浏览器自动化让不可能变可能**

Connectors 提供数百个预置应用连接（零技术配置），不在列表里的用 MCP 接入。浏览器自动化（Mac OS 研究预览）让 Claude 直接操作浏览器截图、点击，适合无 API 的老旧企业应用的重复性操作。

## 可执行建议

1. **立即可做（5 分钟）**：开启 Auto Mode（Shift+Tab 两次），在下一个任务里测试减少多少授权弹窗。
2. **本周可做**：给一个重复性日常任务（如整理邮件）设置 `/goal` + `/loop`，观察 Claude 能否持续运行到完成。
3. **进阶部署**：搭建 VPS + tmux + Channels（Telegram/Discord），实现手机随时派发任务，彻底脱离电脑。
4. **大任务提效**：对需要深度推理的任务（研究报告、代码重构等），使用 `/effort max` 或 Ultra Code，并把计划文件存入项目目录。
5. **探索自动化边界**：在桌面应用"自定义 → 连接器"中浏览现有连接，找到一个可以立即自动化的重复操作。

## 关键术语

| 术语             | 说明                                                     |
| ---------------- | -------------------------------------------------------- |
| Auto Mode        | 自动模式，智能区分安全/危险操作，减少手动授权            |
| `/goal`          | 设置任务完成条件，Claude 必须达标才能停止                |
| `/loop`          | 设置短期定时任务节奏                                     |
| `/routines`      | 设置长期（如每周）定时任务                               |
| `/effort`        | 努力程度滑块，控制推理 token 配额                        |
| Ultra Code       | 最高努力模式，Claude 自行设计并派发多 agent 工作流       |
| Context Rot      | 上下文腐烂，长对话中上下文过大导致判断力下降             |
| Channels         | 内置功能，通过 Telegram/Discord 与 Claude 通信           |
| tmux             | 终端复用工具，用于在 VPS 上保持 Claude Code 会话持久存活 |
| Connectors       | 内置预置连接器，零配置接入数百个第三方应用               |
| MCP              | Model Context Protocol，用于连接无内置 Connector 的应用  |
| Dynamic Workflow | Ultra Code 动态工作流，Claude 为每个任务自行编排多 agent |

## 来源与限制

- 视频时长约 11.5 分钟，偏介绍性，演示为主，不涉及具体代码
- VPS + tmux 方案被作者称为"有点黑客风格"，非官方支持路径
