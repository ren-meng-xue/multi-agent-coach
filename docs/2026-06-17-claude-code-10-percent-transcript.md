当你从 ChatGPT 切换到 Claude Code 时，你被许诺了一个愿景：它会在你所有工具上自动完成重复性工作，处理你的行政事务、运营和外联，在后台默默运行，甚至在你睡觉时也不停歇。等等等等。所以，诚实地问问自己：这真的发生了吗？对大多数人来说，答案可能是否定的。你依然坐在屏幕前，一次一条地输入提示词，做的是和以前一样的工作，只是换了个窗口——一边告诉自己"之后会去搭建那个工作流的"。

这种感觉我太懂了，相信我。但问题不在你，也不在工具本身。问题在于，那些能兑现这个承诺的功能——那些真正能让它在没有你的情况下运行的功能——你还没有打开，或者不知道从哪里开始。好消息是，只需几个小时就能搞定。把正确的几个功能拼在一起，你就可以让工作在你不在桌前时继续进行。

**[00:00:40]**

第一部分：让 Claude 持续运行而不中断。第二部分：让它胜任更大的工作而不降低质量。第三部分：让它完全脱离你的笔记本电脑。最后一部分：开拓你从未想过可能的新领域。我会现场搭建一个示例，让你看到这些功能能多快上手。每一部分都带你更进一步。

## 第一阶段：不中断的自主工作流

**[00:01:10]**

我们先从"不会在任务中途停下"的自主工作流讲起。因为开箱即用的 Claude Code，经常会在任务中途停下来，而且烦人地频繁要求授权。这两件事都是很快就能解决的。

**自动模式（Auto Mode）**

首先是自动模式（Auto Mode）。它能消除 90% 的授权问题。此前你只有两个选项：一是用 `--dangerously-skip-permissions` 自动批准所有操作，二是坐在那里手动批准每一件事，哪怕大多数授权弹窗看起来像是一堆技术术语。自动模式是介于两者之间的中间选项。

现在它已经是 Boris Cherny 使用 Claude Code 完成任务的第一建议，尤其是他所说的"多 Claude 工作流（multi-Claude workflows）"。开启方法很简单：按两次 Shift+Tab，你就能看到自动模式已经打开。它的原理是在后台运行一个分类器（基本上是一个安全检查器），监控 Claude 想做的所有操作。安全的操作会自动放行；只有真正有风险的操作——比如删除文件，或者触及它不该碰的东西——才会停下来请求你的授权。

**[00:02:10]**

**`/goal` 命令：定义"完成"的标准**

我们解决了授权问题，但如何让它运行到真正完成？这就是 `/goal` 命令的用武之地。

设置目标后，Claude 在停止前必须先检查目标是否达成。普通提示词运行一次就停，它自己决定什么时候"完成"了——而这往往会在你定义的"完成"标准被检验之前就提前结束。`/goal` 翻转了这个逻辑：由你来定义"完成"究竟是什么样子。

输入 `/goal` 后，你输入决定任务是否完成的条件。之后 Claude 必须一轮接一轮地持续工作，直到那个条件为真，或者它满足你设定的停止标准。它能做到这一点，是因为有第二个独立的 Claude agent 在目标关闭之前审核工作——它无法自己宣布完成。它会不断产生新的迭代并运行，直到循环结束。

**[00:02:54]**

**结合 `/loop` 和 `/routines` 实现定时任务**

你可以通过把 `/goal` 与 `/loop` 功能结合，让它应用到每天都要做的任务上，变得更强大。`/loop` 让你设置运行的间隔或节奏，然后输入一个包含 goal 条件的提示词。这样就可以：

- 对于**三天以内的短期任务**，结合 `/loop` 使用；
- 对于**更长期的任务**（比如想让它每周运行一次），结合 `/routines` 使用。

举个例子：清理收件箱。Routine 每天或每周触发一次任务，goal 判断任务是否完成——比如"收件箱完全清空才算完成"。Loop 或 Routine 设定节奏，Goal 设定完成标准。而你不需要批准任何非危险操作，因为我们在用自动模式。

## 第二阶段：高质量地完成更大的工作

**[00:03:53]**

现在你可以自主运行了，但对于更大的任务，仅靠开箱即用的 Claude Code，输出质量并不一定好——小任务没问题，但需要大量思考和逻辑推理时呢？这就引出了第二阶段：在不降低质量的情况下完成更大的工作。

质量下降有两个原因：

**第一，上下文腐烂（Context Rot）**

随着对话变长，上下文被撑爆，Claude 无法保留所有信息，判断力下降，无法回忆你告诉它的所有内容。

**第二，推理算力不足**

用 AI 术语说，就是分配给思考和推理的 token 数量不够。如果对一个大问题分配的 token 不够，就得不到好结果。默认情况下使用推理模型时，它会设定一定的 token 数量来推导答案。但我们可以花更多 token 来增加 AI 在思考过程上花费的时间，从而得到更好的结果。

**[00:04:42]**

**`/effort` 努力程度滑块**

两个解决方案都非常简单：努力程度和规划。先说努力程度。Claude Code 内置了努力程度滑块。输入 `/effort`，你可以看到可以为模型使用设置努力等级。这基本上是一个"Claude 对任务思考多努力"的拨盘。可以选择从 low 一直到 max，甚至还有 Ultra Code。

Ultra Code 更进一步：它会产生多个子 agent，用自己的工作流模式来解决你交给它的任务（稍后会详细说）。当然，更高的努力等级（extra high、max、Ultra Code）会消耗大量 token，所以只在真正值得的任务上用。

**[00:05:20]**

**计划要放对地方**

再说规划。我们需要规划，但规划要放在正确的位置。以前的建议是按两次 Shift+Tab，开启计划模式——这对 80% 的事情还不错，但默认情况下，计划文件会被保存到一个项目文件夹之外的临时文件夹。Claude Code 前几次能可靠地读到它，但一旦我们开始压缩上下文，它就会开始失去对最初计划的把握。

**强烈建议把计划存放在项目文件夹内的某个文件或文件夹里**，这样你就可以持续把计划喂给 Claude，让它知道自己在哪一步。

**[00:05:53]**

**Ultra Code 如何运作**

Ultra Code 值得更深入地了解。你可以看到，它有效地为当前轮次请求了一个动态工作流。这意味着：有时候任务太大了，无论计划多好，都装不进一个上下文窗口——而且在一个窗口里也不会快。所以 Claude 现在可以为一个任务编写自己专属的工作流。

它不再在一个窗口里做所有事，而是为那个特定任务设计一个结构化计划。比如，如果我们要对某个主题做深度研究，它会产生一组 agent，分别进入多种工作流模式——Anthropic 列出了六种常用工作流模式。

举例：一个深度研究任务可能扇出成四个 agent，每个研究一条信息或一个来源；第五个 agent 把所有发现合并、综合。还可能有一个质量检查流程：一个 agent 在博客文章里提出论断，另一个负责验证这些论断。每个 agent 都有一个全新的窗口，不会有上下文腐烂；原始目标永远不会丢失，因为没有任何单个 agent 的上下文需要承载太多内容。整个过程由一个编排者协调——也就是你的主对话窗口或主 agent。

**[00:07:00]**

有了这些，你可以在不盯着屏幕的情况下保持高质量输出。既然你不在场，为什么不直接从手机派发任务呢？这样你就能真正从任何地方工作。

## 第三阶段：彻底脱离你的笔记本电脑

**[00:07:11]**

不过经过测试，开箱即用的方案并不完全可行。Claude Code 内置的手机访问方式都有同一个缺陷：

- **远程控制（Remote Control）**：一条命令即可设置，会打开一个 Claude 应用，但断线或不活跃约 10 分钟后，会话就会断开。
- **Channels（频道）**：约 5 分钟即可设置，让你从 Telegram 或 Discord 与 Claude 通信，超级简单——但会话只在你活跃时存活，一旦断开，工作就停了。它不像 Hermes 或 Open Claude 那样能在后台做事，不是我们期望的那个梦。

**但你可以自己搭建。** 有点黑客风格，但有一个变通方案：把 Claude Code 运行在一个**永不休眠**的地方，这样就不再依赖你的笔记本或 PC 保持开机。

**[00:07:52]**

方案：用一台便宜的 VPS（约每月 15 美元），然后把两样东西配对使用：

1. 内置的 **Channels**（连接 Telegram/Discord）
2. **tmux**（一个保持会话持续运行的框架）

tmux 让会话保持存活，这意味着你可以派发任务然后走开。作者做了一整期视频专门讲这个配置，链接在视频描述里。

**[00:08:16]**

这样就实现了：高质量输出 + 从手机派发 + 不达完成标准不停止。

## 第四阶段：你从未想到可能的任务

**[00:08:28]**

最后一阶段的目标是打开你的想象力，让你看到什么是真正可以自动化的。12 个月前，这类事情根本不可能。但现在的限制不是缺少功能——而是你脑子里对"什么是可能的"的认知边界，因为大多数事情现在真的都能做了。

这里快速演示两件事：

**连接器（Connectors）**

首先是连接器，而且完全不需要技术配置。在桌面应用的"自定义"里，点击连接器，你会看到大量预置连接，包括 Claude in Chrome（需要安装 Chrome 扩展）。点击加号、浏览连接，里面有数百个连接，可以接入你日常使用的任何应用。即插即用，用你正常的账号授权一次，然后用普通英语对话。如果某个应用不在列表里，还可以用 MCP 连接几乎任何应用。

**[00:09:20]**

**浏览器自动化（Browser Automation）**

第二个功能更令人印象深刻。Claude 现在可以直接在你的浏览器里操作，**有效地模拟你**——它能点击应用、判断应该点哪个按钮，在浏览器里替你完成任务。目前处于研究预览阶段，macOS 优先，还很早期，速度也比较慢。但作者认为最大价值在于那些老旧笨重的企业应用——存在桌面或浏览器上，根本无法获得 API 访问权限，但你每天都在重复手动操作。

**[00:09:54]**

**演示**

作者的演示提示词是："打开我的浏览器，进入我的 School 社区（school.com/scrapes），读取过去 24 小时内的所有成员帖子，写一个三组分类的优先级列表——今天需要我处理的、可以等待的、值得庆祝的成果。每条给我一句话摘要，不要回复任何人。"

他仍然想亲自去回复（保留人情味），但他想先从消息积压中得到一份摘要。Claude 需要直接进入浏览器，截图并带回结果，再整理成文字摘要。

大约 15 分钟后，Claude 回来说："过去 25 小时内有 9 条成员帖子，这是你的优先级列表：2 人今天需要你的帮助（附摘要），4 人发了信息类帖子，3 人在庆祝他们在社区里的成果。"而且它还直接给出了帖子链接，让你能快速跳转去回复。

**[00:10:53]**

这只是一个简单示例，但希望它打开了你对"什么真正可以自动化"的想象。结合我们提到的另外三个阶段，你现在已经离 Claude Code 最初承诺的愿景近了很多：在你所有工具上运行重复性工作，处理运营、外联、行政，在你睡觉时有效地工作——实际上就是从手机发条消息就能搞定。
