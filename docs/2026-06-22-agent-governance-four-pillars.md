# 代理治理的四大支柱

## 一句话总结

从 POC 到生产，代理治理必须从一开始就内置，而非事后补救——四大支柱（生命周期管理、风险管理、安全、可观测性）决定了 AI Agent 能否安全上线。

## 核心观点

1. **POC 阶段欠下的治理债会拖慢生产部署** — 2~3 周构建的 POC 交付团队后往往需要 8 个月才能上生产，原因是需要重新补齐所有安全与治理环节。治理要从第一天就做，而不是等性能验证完再补。

2. **四大支柱是代理治理的分类框架** — 讲师将代理治理归纳为四个正交维度，每个维度有对应最佳实践：
   - 生命周期管理 → 职责分离
   - 风险管理 → 纵深防御
   - 安全 → 最小权限访问
   - 可观测性 → 全操作审计

3. **生命周期管理 = 职责分离** — 多团队通过 dev/staging/prod 三套环境、版本控制和 CI/CD 流水线分别管理数据和模型变更，每个阶段强制审核，具备回滚能力。

4. **风险管理 = 纵深防御** — 不依赖单一防线，而是叠加多层机制：PII 检测、护栏（Guardrails）、合规控制、数据质量监控，从数据摄入到模型输出全链路覆盖。

5. **安全 = 最小权限访问** — 开发者不应获得所有环境的广泛权限；生产环境必须通过 RBAC、SSO、MFA、细粒度 API 密钥和密钥管理服务实现精确授权，超高权限本身就是风险。

6. **可观测性 = 全程可审计** — 不止是 OpenTelemetry（OTEL）链路追踪，还包括：应用日志、访问日志、数据血缘、告警、合规报告。目标是能追溯代理在运行时使用了哪个工具、访问了什么数据、返回了什么结果。

7. **治理层贯穿整个代理构建生命周期** — 数据准备 → 构建代理 → 评估 → 部署，Databricks 通过 Unity Catalog（统一目录）和 MLFlow 在各阶段提供治理支撑。

## 可执行建议

- **从 Day 1 引入治理**：搭建 dev/staging/prod 三套环境隔离，不要等到要上生产时才补。
- **PII 检测前置**：在数据摄入阶段就加 PII 检测，而非在模型输出侧拦截。
- **用 RBAC 替换广泛权限**：开发者账号只应有开发环境权限，生产权限通过审批流单独授予。
- **部署前对照四支柱清单自查**：
  - 变更能否通过环境安全推广 + 可回滚？
  - 是否有多层防护在生产前拦截问题？
  - 所有数据源是否最小化授权？
  - 能否追踪 Agent 运行时的每个工具调用和数据访问？
- **可观测性超出 OTEL 范畴**：链路追踪只是起点，还需要数据血缘、合规报告、日志访问审计。

## 关键术语

| 术语                             | 说明                                                                          |
| -------------------------------- | ----------------------------------------------------------------------------- |
| 代理治理（Agent Governance）     | 对 AI Agent 生命周期的控制与可见性框架                                        |
| 职责分离（Separation of Duties） | 生命周期管理的核心原则，多团队通过环境隔离分管不同阶段                        |
| 纵深防御（Defense in Depth）     | 风险管理原则，多层防护机制叠加，任一层失效不致全线崩溃                        |
| 最小权限访问（Least Privilege）  | 安全原则，代理和用户只拥有完成其角色所需的最小权限                            |
| PII 检测                         | 个人身份信息（Personally Identifiable Information）检测，风险管理层的核心能力 |
| OTEL / OpenTelemetry             | 开放遥测标准，用于分布式系统（含 LLM 调用）的链路追踪                         |
| Unity Catalog                    | Databricks 统一数据目录，提供跨工作区的数据治理与权限管理                     |
| MLFlow                           | 开源 ML 平台，用于实验追踪、模型注册与部署，Databricks 原生集成               |
| 黄金数据集（Golden Dataset）     | 用于评估代理质量的标准基准数据集                                              |
| 血缘追踪（Lineage Tracking）     | 追踪数据从来源到消费全链路的能力，是可观测性的重要组成部分                    |
