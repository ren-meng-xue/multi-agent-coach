# Multi Agent Coach · 7 Day Ship · 执行计划主索引

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement each day plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在 7 天 × ~9.2h = 64h 内 ship 出 Multi Agent Coach V1，兑现 M2 简历里程碑（多 Agent + 分级记忆 + Reflexion + Clerk 多用户鉴权）。

**Architecture:** Clerk 鉴权（前端 drop-in 组件 + 后端 JWT 校验）+ FastAPI 后端 + LangGraph StateGraph 编排 3 个 Agent（HR / 技术 / Coach）+ MemGPT 范式分级记忆（L0 state / L2 user_profile / L3 star_stories / L4 weakness_tags） + Reflexion 单轮自反思 + pgvector RAG 题库 + Celery 异步写回 + Next.js 16 SSE 流式前端。所有业务数据按 Clerk user_id（VARCHAR）隔离。

**Tech Stack:** Python 3.12 / FastAPI / SQLAlchemy 2.x async / Alembic / Celery 5 / Redis 7 / PostgreSQL 16 + pgvector / LangGraph / OpenAI SDK (gpt-4o + text-embedding-3-small) / Next.js 16 + TS / Tailwind + shadcn/ui / pnpm + uv / Firecrawl / sse-starlette / pytest + pytest-asyncio + httpx / **Clerk（@clerk/nextjs + pyjwt[crypto]）**。

**Spec 来源:** [`../../plans/7-day-ship-plan.md`](../../plans/7-day-ship-plan.md)

## 3. 日常执行规则（每个 day plan 通用）

每个 day 文件包含：
- 头部：本日目标 + 输入 commit + 输出 commit
- Tasks: 按依赖排序的细粒度任务，每个 task 含：
  - **Files:** Create / Modify / Test 的精确路径
  - **Steps:** 失败测试 → 验证失败 → 实现 → 验证通过 → commit 的 5 步循环
- 末尾：本日 EOD commit + 验收清单

**TDD 纪律**（rigid skill）：
- 任何新建函数/模块必须先写失败测试
- 任何 bug 必须先写复现测试
- 每完成一个可独立验证单元立即 commit
- 永远不要 `git add -A` 或 `git add .`，按名字添加文件

**Commit 规范**：`<type>: <subject>`，type ∈ {feat, fix, test, refactor, docs, chore, build}

**Skill 调用约定**：
- 实施前用 `superpowers:test-driven-development` 走 TDD 工作流
- 完成前用 `superpowers:verification-before-completion` 跑验证
- 完成后用 `superpowers:finishing-a-development-branch` 决定整合策略

## 4. 7 天里程碑日程

| Day | 主题 | Hours | 关键交付 | EOD 验证 |
|---|---|---|---|---|
| **D1** | 脚手架 + 数据层 + Clerk 接入 | 10h | 仓库初始化 / Docker / 7 张表（user_id VARCHAR）/ FastAPI /health / Clerk 注册 + env keys | `./dev.sh` 起服务，`curl /health` 返回 ok，`alembic current` 显示 head |

| **D4** | L0 + L2 记忆 + 读策略 | 8h | user_profile CRUD / EMA 更新 / load_memory helper / Celery worker | 第 2 次面试 HR Agent 能引用上次内容（同一 Clerk 用户） |
| **D5** | L3 STAR + L4 弱点 + Reflexion | 8h | STAR 抽取 / 弱点判定 / Reflexion JSON / 写回链路 | 一题答完后 L2/L3/L4 全部有数据 |
| **D6** | 前端 2 页面 + Clerk + 联调 | 11h | 面试房间页 + 复盘抽屉 + 复盘报告页 + `<ClerkProvider>` + 登录/注册页 + middleware 路由保护 + 自动带 JWT | 浏览器跑完整 demo 黄金路径（含登录），两个 Clerk 账号数据隔离验证 |
| **D7** | 测试 + 部署 + Demo + 简历素材 | 9h | pytest 全绿 + README（含 Clerk Setup）+ Demo 视频 + GitHub 公开 | 简历可投 |
